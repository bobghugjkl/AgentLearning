## Week 2：RAG 系统（Day 8-14）

## Week 3：Agent 进阶（Day 15-21）

### Day 15 — 复杂 State + 条件分支
- 多字段 `TypedDict` State 设计
- `Literal` 路由函数
- 三路由图（general/code/high_risk）
- 回边实现循环重试

### Day 16 — Human-in-the-Loop
- HIL 原理：`interrupt()` 暂停 + `Command(resume)` 恢复
- checkpointer 持久化支持跨时间审批
- `MemorySaver` vs `SqliteSaver` vs `PostgresSaver`
- State 初始值的坑（`approved` 默认 False）

### Day 17 — 工具设计规范 + MCP
- 工具设计五原则：description清晰、参数语义化、返回截断、HIL、幂等
- description 是模型选择工具的唯一依据
- MCP 协议：工具标准化，一次写好到处用
- `FastMCP` Server/Client、async/await 异步

### Day 18 — Plan-and-Execute Agent
- Planner → Executor 循环 → Reporter 三节点
- `current_step` 计数器控制循环
- 和 ReAct 的区别：先规划后执行
- 缺点：串行慢、计划固定

### Day 19 — Reflexion 自反思
- 三组件：Actor + Evaluator + Reflection
- 反思内容塞进下一次的 prompt → 越试越聪明
- `max_attempts` 防止无限循环
- 和普通重试的本质区别

### Day 20 — 多 Agent Supervisor
- `create_supervisor` + `create_react_agent`
- `langgraph-supervisor` 包
- 多 Agent 四个坑：不能直接通信、错误传播、token爆炸、调试困难
- Supervisor vs 条件边路由的区别

### Day 21 — Week3 综合项目
- 智能研究助手：Supervisor + 专家 Agent + HIL
- 核心原则：**按需组合，不过度设计**
- HIL 放最外层做统一安全门

## Week 4：LangGraph 进阶 + 生产化（Day 22-28）

### Day 22 — Subgraphs 子图
- 主图嵌套子图：`add_node("name", compiled_subgraph)`
- 共享字段自动传递，子图私有字段主图看不到
- 子图编译时不需要 checkpointer

### Day 23 — Send API 并行执行
- Send API 必须在**条件边函数**里返回，不能在节点里
- `operator.add` 防止并行节点写入覆盖
- Map-Reduce 模式实现

### Day 24 — 长期记忆
- `MemorySaver` 短期 vs SQLite 长期
- LLM 自动提取重要信息（姓名/偏好/目标）
- 跨会话持久化：记忆塞进 system prompt

### Day 25 — LangSmith 追踪

- 查看 token 消耗、延迟、每步输入输出
- `@traceable` 追踪自定义函数
- Agent trace 树形结构分析

### Day 26 — 刹车片检测 Agent（核心项目）

- 11928 张图、75 个型号的特征数据库

### Day 27 — FastAPI 生产部署
- 刹车片 Agent 接入 FastAPI HTTP 接口
- 普通接口 + 流式接口
- LangSmith 追踪接入
- 跨目录 `sys.path` 问题修复

### Day 28 — 缓存优化 + 安全加固

## 踩过的坑

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| xxhash DLL 被 Windows 阻止 | Windows 安全策略 | `pip uninstall xxhash -y && pip install xxhash` |
| pyarrow DLL 被阻止 | Windows 安全策略 | `pip uninstall pyarrow -y && pip install pyarrow` |
| HuggingFace 连不上 | 国内被墙 | `os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"` |
| `langchain.text_splitter` 找不到 | LangChain 1.x 移包 | `from langchain_text_splitters import ...` |
| `EnsembleRetriever` 找不到 | 移到 `langchain_classic` | `pip install langchain-classic` |
| Send API 报错 | 放在节点里而不是条件边 | 改到 `add_conditional_edges` 的函数里返回 |
| HIL interrupt 捕获失败 | 不是普通异常 | 用 `graph.get_state()` 检查 `.next` 字段 |
| 跨目录导入失败 | sys.path 不完整 | 同时加 `day26/` 和 `day26/src/` 两个路径 |

## 面试核心考点速查

### RAG 相关
- RAG 流程：加载→切分→向量化→检索→重排→生成
- chunk_size 选择：300-800，法律/学术文档用大的
- 混合检索：BM25（精确词）+ 向量（语义）+ RRF 融合
- Reranker：CrossEncoder 精排，粗排→精排分工
- HyDE：假答案检索，解决 query-doc 表达不对称
- 幻觉防御：prompt 约束"只基于文档内容回答"

### Agent 相关
- ReAct vs Plan-and-Execute：随时变化用ReAct，步骤可预估用Plan-and-Execute
- HIL：interrupt()暂停 + Command(resume)恢复 + checkpointer持久化
- Reflexion：失败→反思→教训塞进prompt→改进重试
- Supervisor：LLM理解语义分配任务，vs 条件边（代码写死规则）
- 工具设计：description最重要，幂等，返回截断，危险操作加HIL

### 生产化相关
- LangSmith：两个环境变量零代码接入追踪
- 缓存：`set_llm_cache()` 两行，实测300-1000x速度提升
- 安全：API Key认证 + 频率限制 + 输入校验
- 监控：结构化日志 + /health + /metrics + 中间件
- Docker：Dockerfile→Image→Container，解决环境不兼容
- CI/CD：git push → 自动测试 → 自动部署

  PyPDFLoader          → 文档加载

  YOLOv8 OBB    → 旋转目标检测

  LangSmith     → 追踪监控
